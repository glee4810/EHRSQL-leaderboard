What is EHRSQL?

A Practical Text-to-SQL Benchmark for Electronic Health Records(EHRSQL) is a new large-scale, high-quality text-to-SQL dataset for question answering (QA) on electronic health records where the questions are collected from 222 hospital staff, including physicians, nurses, and insurance review and health records teams at a university hospital. The dataset can be used to test three aspects of the QA models: generating queries that reflect the actual needs of different professionals in the hospital, understanding various types of time expressions (absolute, relative, or both), and the capability to abstain its prediction when not confident.

Getting Started

We've built a few resources to help you get started with the dataset.

Download a copy of the dataset (distributed under the MIT license):

  • Training/Dev Set (MIMIC-III)
  • Training/Dev Set (eICU)
  • DB Schema (MIMIC-III / eICU)

To evaluate your models, we have also made available the evaluation script we will use for official evaluation, along with a sample prediction file that the script will take as input. To run the evaluation, use python evaluate-v2.0.py <path_to_dev-v2.0> <path_to_predictions>.

  • Evaluation Script
  • Sample Prediction File (on Dev)

Once you have a built a model that works to your expectations on the dev set, you submit it to get official scores on the dev and a hidden test set. To preserve the integrity of test results, we do not release the test set to the public. Instead, we require you to submit your model so that we can run it on the test set for you. Submission Tutorial

EHRSQL uses Precision, Recall, and F1 scores for its evaluation metrics. The final output should be either SQL queries or "NULL" if the predictions are abstained.
